=================================================
      SUPABASE & VERCEL DEPLOYMENT GUIDE
=================================================

Since you want to host this project online (using Vercel) and use a real cloud database (Supabase), follow these exact steps!

I have already updated your `prisma/schema.prisma` file to use PostgreSQL instead of SQLite. Now you just need to create the database and connect it.

STEP 1: CREATE YOUR SUPABASE DATABASE
-------------------------------------------------
1. Go to https://supabase.com/ and create a free account.
2. Click "New Project" and choose your organization.
3. Give your project a name (e.g., "EventApp").
4. **IMPORTANT:** Create a strong Database Password and save it somewhere! You will need it in Step 3.
5. Choose a region close to you, and click "Create new project".
6. Wait 1-2 minutes for the database to finish provisioning.

STEP 2: GET YOUR CONNECTION STRINGS
-------------------------------------------------
To connect your app safely, you need two links from Supabase.
1. On your Supabase dashboard, click the "Settings" gear icon (bottom left).
2. Click "Database" in the settings menu.
3. Scroll down to the "Connection string" section.
4. Select the "URI" format.

You will see a link that looks like this:
postgresql://postgres.[your-id]:[YOUR-PASSWORD]@aws-0-[region].pooler.supabase.com:6543/postgres

Notice the port number "6543" - this is your Pooler URL. 
Also notice "5432" - this is your Direct URL.

STEP 3: UPDATE YOUR .ENV FILE
-------------------------------------------------
1. Open your `.env` file in VS Code (`e:\MedWeb\event-app\.env`).
2. Replace your current `DATABASE_URL` with the new ones from Supabase. Make sure to replace `[YOUR-PASSWORD]` with the password you made in Step 1.

Your `.env` file should look like this:

# This is the "Transaction Pooler" string (using port 6543 & pgbouncer)
DATABASE_URL="postgresql://postgres.[id]:YOUR_PASSWORD@aws-0-eu-central-1.pooler.supabase.com:6543/postgres?pgbouncer=true"

# This is the "Direct Connection" string (using port 5432)
DIRECT_URL="postgresql://postgres.[id]:YOUR_PASSWORD@aws-0-eu-central-1.pooler.supabase.com:5432/postgres"

# Your email settings (Leave these as they are)
EMAIL_USER="your-email@gmail.com"
EMAIL_PASS="your-app-password"

STEP 4: PUSH YOUR TABLES TO SUPABASE
-------------------------------------------------
Now that your app is connected to Supabase, you need to create the tables in the empty cloud database.

1. Stop your local server if it is running (click in the terminal and press Ctrl + C).
2. Run this command to push the tables to Supabase:
   npx prisma db push

3. Run the seed script to create your Super Admin account in the cloud database:
   node seed.js

Now your cloud database is fully set up!

STEP 5: UPLOAD TO GITHUB
-------------------------------------------------
1. Open your terminal in VS Code.
2. Run these commands one by one to upload your code to your new GitHub repo:
   git init
   git add .
   git commit -m "First commit: Event App ready for Supabase"
   git branch -M main
   git remote add origin https://github.com/YOUR-USERNAME/YOUR-REPO-NAME.git
   git push -u origin main

*(Make sure to replace the GitHub link with your actual repo link!)*

STEP 6: DEPLOY TO VERCEL
-------------------------------------------------
1. Go to https://vercel.com/ and log in with your GitHub account.
2. Click "Add New..." -> "Project".
3. Import your new GitHub repository.
4. **CRITICAL STEP:** Before clicking Deploy, open the "Environment Variables" section!
5. Add all 4 variables from your `.env` file:
   - DATABASE_URL
   - DIRECT_URL
   - EMAIL_USER
   - EMAIL_PASS
6. Click "Deploy". 

Congratulations! Vercel will build your app, and in about 2 minutes, you will get a live URL that you can share with your attendees!
