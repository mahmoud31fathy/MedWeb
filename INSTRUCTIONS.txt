=================================================
          EVENT APP - SETUP & INSTRUCTIONS
=================================================

1. ACCESSING THE APPLICATION
-------------------------------------------------
Since the server is already running, you can access the application using your web browser.

- Public Registration Page: 
  http://localhost:3000
  (Attendees go here to register for the event)

- Admin Portal / Dashboard: 
  http://localhost:3000/admin/login
  (You and your admins go here to scan QR codes and view stats)


2. ADMIN LOGIN CREDENTIALS
-------------------------------------------------
I have created a default "Super admin" account for you. Use these credentials to log in:

  Email:    admin@event.com
  Password: admin123

Once logged in, you can create additional "admin" accounts from the "Admins" tab.


3. HOW TO ENABLE QR CODE EMAILS
-------------------------------------------------
To allow the system to send emails with the QR code to people who register, you need to configure your email credentials.

Step 1: Open the file named ".env" located in "e:\MedWeb\event-app\.env"
Step 2: Add these two lines at the bottom of the file:
        EMAIL_USER="your-email@gmail.com"
        EMAIL_PASS="your-app-password"

Note for Gmail users:
Because of security settings, you cannot use your regular Gmail password. You must generate an "App Password".
1. Go to your Google Account -> Security.
2. Turn on 2-Step Verification if you haven't already.
3. Search for "App Passwords" in the search bar.
4. Create a new App Password (call it "EventApp") and paste the 16-character code into your EMAIL_PASS variable.


4. SCANNING QR CODES AT THE DOOR
-------------------------------------------------
1. Open the Admin Portal on a mobile phone or laptop.
2. Log in with the admin credentials.
3. Click on the "Scan QR" tab.
4. Grant the browser permission to use the camera.
5. Point the camera at the attendee's QR code. The system will automatically verify it and mark them as "Attended".

5. HOW TO ACCESS THE APP ON YOUR PHONE (FOR SCANNING)
-------------------------------------------------
To scan QR codes with your phone, you need to open the app on your phone's browser. However, modern phone browsers require a secure connection (HTTPS) to give permission to use the camera. 

The easiest way to get an HTTPS link while the app is running on your computer is to use a free tool called "localtunnel":

1. Make sure your server is currently running (you started it with "npm run dev").
2. Open a NEW, separate terminal window in VS Code (keep the first one running!).
3. Type this command and hit Enter:
   npx localtunnel --port 3000

4. After a few seconds, it will give you a secure link that looks something like this:
   "your url is: https://some-random-words.loca.lt"

5. Type that exact "https://..." link into your phone's web browser (Safari or Chrome).
6. Note: The first time you open the link, it might show a warning page asking for a "Tunnel Password" or "Endpoint IP". Just click the "Click to Continue" button to bypass it.
7. Go to the admin login page on your phone, log in, and click the "Scan QR" tab. Your phone will ask for camera permissions, and you are ready to scan!

6. HOW TO PUBLISH THIS PROJECT TO THE INTERNET
-------------------------------------------------
This project is built using Next.js, which means both your Frontend (the website screens) and Backend (the API, database, and emails) are combined into this ONE single project folder. You do not need to publish them separately!

However, because this project currently uses an "SQLite" database (which saves data to a local file named 'dev.db'), you CANNOT host it on "Serverless" platforms like Vercel or Netlify out of the box. Those platforms reset local files constantly, meaning your registered attendees would be erased!

Here are the best ways to publish this project:

OPTION A: Railway.app or Render.com (Recommended & Easiest)
1. Push your entire project code to a GitHub repository.
2. Create an account on Railway.app or Render.com.
3. Create a new "Web Service" and link it to your GitHub repository.
4. IMPORTANT: You MUST configure a "Persistent Volume" (a permanent virtual hard drive) and map it to your database folder. This ensures your SQLite database file isn't deleted when the server restarts.
5. In the platform's settings, add your Environment Variables (EMAIL_USER, EMAIL_PASS, etc.).
6. The platform will automatically build and host your app and give you a live HTTPS link!

OPTION B: A Virtual Private Server (VPS) like DigitalOcean, AWS EC2, or Hostinger
If you want full control, you can rent a Linux server.
1. Rent a basic Ubuntu server (usually $4 to $6 a month).
2. Install Node.js, Git, and Nginx on the server.
3. Copy your code onto the server.
4. Run `npm install` and `npm run build`.
5. Use a tool called "PM2" (`npm install -g pm2`) to keep your app running forever in the background: `pm2 start npm --name "event-app" -- start`.
6. Use Nginx to route your domain name to the app and set up a free SSL certificate using Certbot.

OPTION C: Vercel (Requires Changing the Database)
If you really want to use Vercel (which is free and made specifically for Next.js):
1. You must replace the local SQLite database with a cloud database like PostgreSQL (you can get a free one from Supabase or Neon).
2. Change `provider = "sqlite"` to `provider = "postgresql"` in your `prisma/schema.prisma` file.
3. Update your `DATABASE_URL` in `.env` to the new Postgres link.
4. Push your code to GitHub and connect it to Vercel. Vercel will host the website, and Supabase will safely host the data.
